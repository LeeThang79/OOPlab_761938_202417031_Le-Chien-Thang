● If you create a constructor method to build a DVD by title then create a constructor method to build
a DVD by category. Does JAVA allow you to do this?
    + Answer: No, Java does not allow this. Since both title and category are of type String, two constructors each taking a single String parameter would have the exact same method signature. Java cannot distinguish which constructor to call based solely on the variable name. To overload a constructor, you must have a different number of parameters, different data types, or a different order of data types.

● What do you prefer in this case?
    + Answer: I prefer using variable arguments because it is more flexible and results in cleaner code. Arrays require you to explicitly declare and initialize an array in the main method before passing it, variable arguments allow you to pass any number of arguments directly, separated by commas.

● Is JAVA a Pass by Value or a Pass by Reference programming language?
    + Answer: Java is a Pass-by-Value programming language.

● After the call of swap(jungleDVD, cinderellaDVD) why does the title of these two objects still remain?
    + Answer: The titles remain unchanged because Java passes the copy of the bits representing the reference. When the swap() method is called, Java does not pass the actual objects or the original references; instead, it creates copies of those references. The swap only occurs between the local copies inside the method's scope, leaving the original references in the main method pointing to their original objects.

● After the call of changeTitle(jungleDVD, cinderellaDVD.getTitle()) why is the title of the JungleDVD changed?
    + Answer: Although Java passes a copy of the reference, this copy still points to the same underlying object on the Heap memory. Therefore, when the setTitle() method is invoked through the copied reference, it modifies the actual data of the original object. This is why the change is reflected outside the method.
